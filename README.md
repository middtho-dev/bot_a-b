# 🤖 EASY WAY Bot

Telegram-бот для команды: учёт активности, чек-ин, EOD, события продаж/логистики, отчёты и алерты неактивности.

---

## ✅ Статус по ТЗ (Week 1–4)

Ниже — фактическое покрытие в текущем коде:

### Week 1 — активность
- Учёт количества сообщений сотрудника в рабочих чатах.
- Фиксация первой/последней активности за день.
- Дневной и недельный отчёты в админ-чат.

### Week 2 — check-in / EOD
- Кнопка чек-ина по расписанию.
- Ручная команда `/checkin`.
- EOD-форма в 4 шага в личном чате с ботом.
- Напоминание и список, кто не сдал EOD.

### Week 3 — Sales / Logistics
- `/sale` (только для роли sales в чате продаж).
- `/shipment` (только для роли logistics в чате логистики).
- KPI-блок в недельном отчёте.

### Week 4 — контроль неактивности
- Проверка каждые 10 минут.
- Учитываются рабочие часы и порог неактивности.
- Исключается роль `finance`.
- Лимит: до 2 алертов в день на сотрудника.
- Учитывается персональный график сотрудника (по дням недели или 2/2).

> ⚠️ Важно: «всё работает» подтверждается только в реальном Telegram-окружении (бот в чатах + права + корректные chat_id). В репозитории выполняются статические проверки, но не end-to-end тесты с Telegram API.

## 🧩 Ключевые возможности

- 🛠 Полностью русскоязычный интерфейс меню и уведомлений.
- 👮 Админ-меню `/admin` с кнопками тестов и настройками runtime.
- ⚙️ Настройка чатов/времени/порогов из бота (без обязательного редактирования `.env`).
- 👥 Мастер добавления сотрудника через меню:
  - роль → user_id → имя → username;
  - бот автоматически пытается подтянуть имя и username по user_id.
- 🗑 Удаление сотрудников из админ-меню (кроме owner).
- 📅 Индивидуальные графики сотрудников:
  - по дням недели (переключатели ⚪/✅),
  - сменный режим 2/2 со стартовой датой.
- 📄 Просмотр хвоста логов и отправка файла логов из админ-меню.

## 👮 Команды

### Для всех
- `/myid` — показать ваш `user_id`.
- `/chatinfo` — показать данные текущего чата (id, тип, название).

### Для сотрудников
- `/checkin` — ручной чек-ин.
- `/eod` — запуск EOD (в группе бот направит в личный чат).
- `/sale` — форма продаж (только sales + чат продаж; при неверной роли бот явно пишет причину).
- `/shipment` — форма логистики (только logistics + чат логистики; при неверной роли бот явно пишет причину).

### Для owner (`OWNER_IDS`)
- `/status` — состояние бота + runtime-настройки.
- `/report today` — дневной отчёт вручную.
- `/week` — недельный отчёт + KPI вручную.
- `/add_employee role=sales|logistics|finance|general` — добавить сотрудника reply-командой.
- `/set_checkin_time HH:MM`
- `/set_eod_time HH:MM`
- `/set_report_time HH:MM`
- `/export_csv` — выгрузка активности за текущий день.
- `/export csv` — совместимый старый формат команды.
- `/admin` — открыть админ-меню.
- `/admin_test` — алиас `/admin`.

## ⚙️ Runtime-настройка через `/admin`

В админ-меню доступны:
- Тестовые отправки (чек-ин, EOD, отчёты, проверка неактивности).
- Меню переменных с карточками параметров:
  - описание;
  - текущее значение;
  - источник значения;
  - статус: ⚪ не задано / ⭐ установлено.
- Меню сотрудников:
  - список сотрудников;
  - добавление сотрудника в интерактивном режиме;
  - удаление сотрудника;
  - настройка персонального графика (дни недели или 2/2).
- Логи:
  - последние 50 строк;
  - отправка файла логов.

## 🧪 Быстрая самопроверка после запуска

1. В ЛС боту: `/myid`.
2. В рабочем чате: `/chatinfo`.
3. В ЛС owner: `/admin` и кнопки:
   - «Отправить чек-ин»
   - «Кто не чек-ин»
   - «Отправить напоминание о EOD»
   - «Кто не сдал EOD»
   - «Отправить дневной отчёт»
   - «Отправить недельный отчёт + KPI»
4. Проверить, что сообщения ушли в нужные чаты.

## 🚀 Установка с нуля (Ubuntu)

```bash

cd /opt
git clone https://github.com/middtho-dev/bot_a-b.git
cd /opt/bot_a-b

python3 -m venv .venv
source .venv/bin/activate
python -m pip install --upgrade pip
pip install -r requirements.txt

nano .env

```

## 📝 Примечания

- Боту нужны права на удаление сообщений в группах для «чистого чата».
- Без корректно заданных `chat_id` планировщик не сможет отправлять сообщения по расписанию.
- При анонимном режиме администратора Telegram может скрывать `from_user.id`; owner-команды могут не срабатывать.
